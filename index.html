    Proyecto Chimera

    División Chimera — Comunicado Interno

    "Habéis sido seleccionados para participar en un proyecto confidencial."

    Bienvenidos al Backend Engineering Program. Durante este curso construiréis un backend profesional completo como ingenieros junior de la División Chimera.

    La misión

    Objetivo

    Desarrollar el backend del Proyecto Chimera siguiendo arquitectura profesional real.

    Nuestra organización necesita ingenieros capaces de construir:

    • APIs profesionales
    • Sistemas backend escalables
    • Infraestructuras seguras

    Qué construiréis

    Al finalizar el curso tendréis un backend completo con:

    Componente Tecnología
    API REST TypeScript + Node.js + Express
    Base de datos PostgreSQL
    Autenticación JWT
    Infraestructura Docker

    Stack tecnológico

    • TypeScript

      Lenguaje principal del proyecto. Tipado estático sobre Node.js para un código más robusto y mantenible.

    • Node.js

      Runtime backend. Ejecuta TypeScript compilado en el servidor con alto rendimiento.

    • Express

      Framework HTTP minimalista para construir la API REST. Rutas, middlewares y controladores.

    • PostgreSQL

      Base de datos relacional. Almacena y gestiona todos los datos del proyecto con SQL.

    • Docker

      Contenerización de la app y la base de datos. El entorno es reproducible en cualquier máquina.

    • Claude Code / Codex

      IA asistida integrada en el flujo de desarrollo. Generación de código, debugging y documentación.

    Plan de formación — Metodología Scrum

    El programa se organiza en 8 sprints progresivos. Cada sprint tiene un MVP entregable y criterios de aceptación claros. Al final de cada sprint tendréis algo funcional que podéis ejecutar y demostrar.

    Sprint Nombre MVP
    1 Setup & Primer Endpoint App Express con GET /health
    2 API REST en Memoria CRUD completo sin BD
    3 Persistencia con PostgreSQL CRUD con BD real
    4 Arquitectura Software Refactor a capas
    5 Autenticación JWT Login/Register + rutas protegidas
    6 Testing de API Suite de tests automatizados
    7 Docker & DevOps App + BD en docker-compose
    8 MVP Completo Backend listo para producción

    Paso 1 — Investigación previa

    Antes de la primera sesión

    Investigad los siguientes conceptos. No hace falta código, solo entender las ideas.

    Modelo cliente-servidor

    ¿Qué ocurre exactamente cuando escribes una URL en el navegador y pulsas Enter? Identificad quién es el cliente y quién es el servidor en ese proceso.

    Qué es una API

    Buscad ejemplos reales: Netflix, Spotify, la app del tiempo de vuestro móvil. ¿Cómo obtienen sus datos? ¿Qué papel juega una API en esa comunicación?

    HTTP — métodos y códigos de estado

    • Métodos: GET, POST, PUT, DELETE — ¿para qué sirve cada uno?
    • Códigos de estado: ¿qué significa cada rango? (2xx, 4xx, 5xx)

    JSON

    Aprended la sintaxis básica. ¿En qué se diferencia un objeto JavaScript de JSON? Es el formato de comunicación que usaréis durante todo el curso.

    Bases de datos relacionales

    Sin entrar en SQL todavía: ¿qué es una tabla? ¿qué es una fila? ¿cómo se relacionan dos tablas entre sí?

    Listos para despegar

    Con estas bases claras, estaréis preparados para arrancar el Sprint 1. Consultad la planificación completa para ver todos los objetivos.
